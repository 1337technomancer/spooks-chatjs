Beginners guide.

Starting your own chat directory/channel/world.

Simply add a word to the url. Example: http://this.spooks.me/example or http://example.spooks.me/

To the bottom of your screen you'll see the input box. Go ahead and just click in that and type in your unimportant opinions.
In the input box, you'll have the capability to type in /commands.

Crash course:

Commands:

This is an incomplete list of all of the chat commands.
/nick = This allows you to choose your own unique name. SYNTAX: /nick username
/register = This attaches a password to your nickname so that it is only usable by you. SYNTAX: /register password email@address.com
/verify = Immediatly after registration, you should recieve an email (located in Junk/Spam folder) with instructions. As of 25th of July 2014 just type /verify password
/login = If you have been logged out by any means (usually a nick change) you can log in with this command. SYNTAX: /login password username
/mute = Turns chat sound off.
/unmute = Turns the chat sound back on.
/pm = With this command you can send private messages to your friend. Nobody else can view these messages. SYNTAX: /pm username|message
/unregister = This unattaches your password to your nickname so that it is usable to all. SYNTAX: /unregister
Press F13 to send a bug ticket.

Decorative Commands:

Decorative commands are just for show. It will make you feel like the prettiest girl at the ball.
/me = Typing this command will show an action of some kind. SYNTAX: /me walks outside
/color = This will change the colour of the text following your username. SYNTAX: /color tomato
Read more about colors here: http://en.wikipedia.org/wiki/Web_colors
/font = The chat system allows for the stylizing of your font. SYNTAX: /font Gabriela
All available fonts are listed here: http://www.google.com/fonts
/flair = Meaning to add colors and fonts to your name. SYNTAX: /flair $font|#colorusername
Example: /flair $Gabriela|#F00000Jason

What is /whoami ?
After using this command you'll see some confusing information like this:
You are MYNAME with role basic with access_level 0 with ip xx.xx.xx.xx
Basically you're broken down into two groups.
Your "role" and your "access_level"

Your role defines you as:
muted (Unable to be heard)
basic (Can talk)
mod (Able to change the topic and give access to other users only up to a level below their own.)
admin (Able to channel_ban users. If you own a channel, this will be your default.)
super (Able to globally ban users and everything else)

Your access_level defines you as the level your role is:
If you're a mod with level 3, you cannot ban a mod with level 2. Also you can give another user the power of modship but only up to level 4 or lower.

How do people put codes in this website?

They have knowledge of either html javascript or css and depending how deep you go, maybe other languages.
Then they go to github where spooks is hosted and make a pull request; adding new code to the old code.
Only certain people can accept the request and push the updates.

And github is a...?

It's a hosting website for code.
Usually for developers to get help.
We use it as a third party to inject code directly into the server.
https://github.com/codefactor/spooks-chatjs
That is spooks.
After the request is pushed, it is up to a higher admin, like me, to reset the chat, so the code is correctly injected.
There's a few steps to all of this, but it helps security.
I get an email for every pull request, so I know what to merge and when to reset.
